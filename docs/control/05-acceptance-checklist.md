# 生活管理 App 驗收清單

狀態：正式控制文件  
任何功能或版本不得只以「程式能跑」作為完成標準。

## A. 產品定位

- [ ] 變更屬於生活管理 App，而非 PMS。
- [ ] 功能名稱一般人一眼看懂。
- [ ] 沒有重新使用「先放著」「需要你記住的事」「承接責任」。
- [ ] 沒有把所有事情壓成同一種待辦。
- [ ] 保養項目、處理事件卡與史略角色清楚分開。

## B. 核心流程

- [ ] 簡單事項可以快速完成。
- [ ] 複雜修理、工程或辦理事項可持續追蹤。
- [ ] 處理進度可追加，不會覆蓋前一筆。
- [ ] 尚未真正完成的事情不能直接進入完成史略。
- [ ] 結案後完整過程可回顧。
- [ ] 取消時保留原因。

## C. 週期與時間

- [ ] 每日、每週、每月、每季、每半年、每年各自計算。
- [ ] 延遲完成不改變原時間尺度。
- [ ] 月底、跨年與閏年有測試。
- [ ] 暫停、恢復與結束狀態正確。
- [ ] 同一期不會重複產生提醒。
- [ ] 使用者主動重新排程時結果符合預期。

## D. 階段性重點

- [ ] 觸發條件與一般固定週期分開。
- [ ] 達到門檻時先形成提醒，不直接形成完成紀錄。
- [ ] 使用者開始處理後才建立事件卡。
- [ ] 年限、里程、次數或其他條件單位清楚。
- [ ] 條件修改不破壞既有史略。

## E. 資料安全

- [ ] 舊 JSON 可讀。
- [ ] 缺少欄位有安全預設。
- [ ] 未知 enum 不會使整組資料失效。
- [ ] 一筆錯誤不會讓全部資料消失。
- [ ] 頂層 JSON 損壞不會被視為正常空資料。
- [ ] 異常時所有寫入被阻止。
- [ ] 原始資料已建立不可變備份。
- [ ] 備份失敗時不繼續遷移或寫入。
- [ ] 遷移可重複執行且不產生重複資料。
- [ ] 遷移前後筆數與關聯已比對。
- [ ] 有可實際執行的回復方案。

## F. UI 與操作

- [ ] 首頁 3 秒內看懂用途。
- [ ] 目前要處理、處理中、即將到期與最近完成有清楚主次。
- [ ] 不顯示內部 ID。
- [ ] 沒有看似可操作但無作用的控制項。
- [ ] 空狀態說明下一步。
- [ ] 表單欄位名稱直接且容易理解。
- [ ] 錯誤訊息說明發生什麼與下一步。
- [ ] 資料保護模式明確阻止修改。
- [ ] 主要操作可在手機單手完成。

## G. 視覺品質

- [ ] 第一眼像正式產品，不像工程測試頁。
- [ ] 字級、間距、卡片與按鈕有一致系統。
- [ ] 不同狀態有明確視覺差異。
- [ ] 不以大量相同圓角卡片堆疊所有資訊。
- [ ] 表單不擁擠。
- [ ] 空畫面與錯誤畫面有完整設計。
- [ ] 已在實際手機尺寸檢查。
- [ ] 專案負責人看過畫面並批准。

## H. 安全邊界

- [ ] 未知風險視為高風險。
- [ ] 高風險內容不提供 DIY 步驟。
- [ ] 健康內容不做診斷或藥量建議。
- [ ] 專業處理建議清楚但不製造恐慌。

## I. 技術驗收

- [ ] `flutter analyze` 通過。
- [ ] `flutter test` 通過。
- [ ] `flutter build web --release` 通過。
- [ ] 需要時完成 iOS build。
- [ ] 新增邏輯有單元測試。
- [ ] Repository 有資料相容測試。
- [ ] 主要操作有 Widget／流程測試。
- [ ] 沒有超出 PR 宣告範圍的變更。

## J. PR 與文件

- [ ] PR 說明修改目的。
- [ ] PR 列出不修改範圍。
- [ ] PR 說明資料與 UI 影響。
- [ ] 控制文件已同步更新。
- [ ] 變更紀錄已新增條目。
- [ ] CI 實際通過後才合併。

## 最終判定

任何一項高風險資料、安全或產品身分檢查未通過，整個功能不得驗收。
