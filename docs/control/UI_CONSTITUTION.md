## 1. 最高視覺依據

上述圖片為董事長核准的 UI v3 最低驗收標準。任何頁面不得以開發者個人喜好或新的視覺風格取代它。參考圖中的 PMS 名稱只是視覺稿標題，不得導入 PMS 產品定位、任務績效、完成率或焦慮設計。

產品正式定位仍為 Life Management：Item 是 Root；Task 是提醒；WorkCase 是案件；WorkCaseClosure 是正式結案；History 是唯讀投影。

## 2. 參考圖強制規格

- 背景：暖白，不用大面積深色 Hero。
- 主色：深藍文字、亮藍操作，狀態色只用於狀態。
- 字級：標題 20–22pt、次標 16–18pt、內文 14–16pt、輔助 12–14pt。
- 間距：8／12／16／24／32；首屏不得用無意義空白拉開資訊。
- 圓角：12–16pt，使用 1px 輕邊框與低陰影，不用笨重懸浮卡。
- 密度：同一手機首屏必須清楚表達頁面用途、主要狀態與第一個操作。
- 層級：一頁一個明確主操作；次要操作降階，不與主按鈕搶層級。
- 圖示：使用一致的線性圖示與 40–44pt 容器；Badge 必須簡短、低壓力。
- 導覽：保留生活總覽、生活項目、新增、史略、設定五入口及原順序。

## 4. 驗收證據

每一頁都必須保留目前版本截圖，與核准參考圖並排比對，並記錄：

1. 字級與文字層級。
2. 資訊密度與首屏容量。
3. 留白與區塊間距。
4. 卡片高度、比例、邊框與陰影。
5. 配色、Icon 與 Badge。
6. 主要與次要操作層級。
7. SafeArea、320×568、390×844、428×926、200% 文字與 Reduce Motion。

沒有逐頁比對證據，或任一頁未達最低標準，PR #242 必須維持 Draft。自動測試與 CI 全綠不能取代董事長的視覺核准。

## 5. 禁止事項

禁止自行設計、只調 Theme 就宣告完成、新增視覺風格、修改功能或欄位、變更五入口、改動 Domain／Schema／Migration／Repository／Drift／資料流程／正式生命週期，或建立平行流程。

## 6. 解除 Draft Gate

只有在指定頁面全部附上未加工真機截圖、逐頁比對完成、自動化與 build Gate 全綠，且董事長明確批准後，才可解除 Draft。
