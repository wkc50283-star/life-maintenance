# 生活維護管家產品憲法與能力地圖

## 1. 產品定位

生活維護管家不是一般保養 App，也不是單純待辦清單。

它是一套幫助使用者長期承接責任、留下可信履歷的系統。使用者不只是完成一個任務，而是在為設備、車輛、房屋部位、文件、保險、健康檢查、長輩照護、寵物照護等需要長期追蹤的對象，建立可回顧、可交代、可延續的紀錄。

PMS 預防性維保只是第一個應用場景。產品的底層能力是把「需要被記住、提醒、執行、紀錄、追蹤」的責任，整理成長期履歷。

## 2. 使用者價值

- 幫你記住重要的事。
- 在需要處理時提醒你。
- 在處理後留下可信的紀錄。
- 長期形成可查詢、可回顧、可交接的履歷。

使用者不需要記得每一件事發生過什麼，也不需要把維修、保養、到期、費用與店家資訊散落在聊天紀錄或腦中。App 的價值是把這些責任整理成穩定的生活維護履歷。

## 3. 產品哲學

### 每一份需要長期承擔的責任，都值得留下履歷

只要一件事需要反覆提醒、持續追蹤、留下處理結果，它就不只是一次性任務，而是一份責任。責任應該留下履歷。

### 不替使用者定義世界

產品不預設使用者只能管理家電、車輛或房屋。使用者可以把需要長期照顧的對象放進系統，例如文件、保險、健康檢查、長輩照護、寵物照護等。

### 架構開放，入口簡單

底層架構允許不同類型的維護對象與紀錄逐步擴充，但第一層入口必須保持簡單。使用者看到的是清楚的建立、提醒、完成、紀錄與履歷流程，而不是複雜分類或專業術語。

### 自由的是內容，一致的是流程

使用者管理的內容可以很自由，但流程必須一致：

```text
建立 → 提醒 → 執行 → 紀錄 → 履歷
```

一致的流程讓不同責任都能被系統承接，也讓資料能長期整理成可信履歷。

### 安全高於便利

產品不為了讓流程看起來完整而提供高風險 DIY 教學。任何高風險或未知風險項目，都必須優先保護使用者安全。

## 4. 核心流程

```text
建立 → 提醒 → 執行 → 紀錄 → 履歷
```

### 建立

建立一個名稱與提醒，承接一份需要長期追蹤的責任，例如冷氣、機車、租屋合約、保險、健康檢查或照護事項。

### 提醒

依照週期、到期日或限期處理需求提醒使用者。提醒不是目的，提醒是讓責任不被遺忘。

### 執行

使用者依照安全邊界處理任務。低風險任務可以提供保養步驟；中風險任務只提供靜態提醒；高風險與未知風險不提供 DIY 步驟。

### 紀錄

任務完成後留下保養維修紀錄，包含處理內容、費用、店家、零件、備註、結果等既有欄位可承接的資訊。

### 履歷

系統把長期紀錄整理成履歷，讓使用者能回顧過去處理了什麼、花了多少、找過誰、結果如何，以及後續需要追蹤什麼。

## 5. 安全紅線

- 高風險與未知風險不提供 DIY 教學。
- 未知風險一律視為高風險處理。
- 高風險項目只能記錄、提醒、形成履歷，並引導尋求合格專業人員協助。
- 不提供電力、瓦斯、煞車、冷媒、結構施工、高壓設備、高溫設備等 DIY 維修步驟。
- 健康或照護相關內容只能做提醒與紀錄，不做診斷、不給醫療建議、不調整藥量。

安全紅線不會因維護對象範圍擴大而放寬。產品可以擴充能記錄的責任，但不能擴充到教使用者執行高風險處置。

## 6. 能力地圖

### 建立

- 建立一個可追蹤的名稱與責任。
- 建立到期提醒。
- 建立保養維修紀錄。
- 未來可承接更多責任類型，但入口仍應保持簡單。

### 提醒

- 根據週期、到期日或限期處理需求形成任務。
- 逾期不改變原本週期基準。
- 提醒只負責讓責任浮上檯面，不取代使用者判斷。

### 執行

- 低風險任務可提供保養步驟 checklist。
- 中風險任務只顯示靜態提醒或注意事項。
- 高風險與未知風險不顯示 DIY steps。
- 完成流程應回到任務完成與紀錄建立，不在步驟 UI 內直接處理資料儲存。

### 紀錄

- 保養、維修、更換、到期處理、限期處理都應能抽象成保養維修紀錄。
- 紀錄可承接處理內容、費用、店家、零件、備註與結果。
- 紀錄的目的是形成可信的長期事實，而不是只關閉一個提醒。

### 履歷

- 履歷由紀錄整理而成，不要求使用者手動維護另一套資料。
- 使用者可以從履歷看到時間、對象、紀錄類型、處理內容、結果與相關補充欄位。
- 履歷是產品長期價值的主要呈現。

### 資料可靠性

- 本機資料讀取需要能承受損壞或不相容資料。
- 儲存格式要保持穩定，模型變更必須謹慎。
- 不為了短期 UI 方便破壞資料格式。

### 未來責任中心

未來產品可以從生活維護延伸為更通用的責任中心，但仍必須維持相同流程：

```text
建立 → 提醒 → 執行 → 紀錄 → 履歷
```

可延伸方向包括文件、保險、合約、健康檢查、長輩照護、寵物照護、家庭共同責任等。這些延伸應逐步進行，不應讓第一次使用變複雜。

## 7. 不做什麼

- 不做社群。
- 不做聊天。
- 不做萬能 AI 助手。
- 不做高風險 DIY 教學。
- 不做大量模板先行。
- 不做會讓第一次使用變複雜的功能。
- 不用專業術語壓過使用者理解。
- 不為了功能完整而犧牲安全邊界。
