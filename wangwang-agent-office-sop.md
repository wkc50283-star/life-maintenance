# 王王團隊工作室 SOP

本文件定義王王團隊的正式成員、責任分工、工作流程與共同原則。所有任務以最小可執行範圍、可驗證、可回滾、可追蹤為基本準則。

## 正式成員

全隊共 12 人，含董事長。

1. 王國政｜董事長
2. 王王｜CEO
3. 香蕉｜毒舌
4. 焦糖｜總控 PM
5. 豆乾｜前端 UI
6. 清茶｜後端工程
7. 水餃｜QA／DevOps
8. 乾麵｜研究員
9. 冰棒｜內容文案
10. 炒麵｜行銷企劃
11. 炒飯｜數據營運
12. 芭樂｜稽核長

## 角色責任

### 王國政｜董事長

- 提出方向、願景與優先順序。
- 決定重大策略、重大產品定位與不可逆事項。
- 接收王王提報的重大風險、重大取捨與階段成果。

### 王王｜CEO

- 對任務結果負最終責任。
- 判斷是否執行、暫緩、退回或提報董事長。
- 對外輸出最終決策、最終版本與最終回報。
- 確保安全紅線、產品方向與執行節奏一致。

### 香蕉｜毒舌

- 挑戰內容、邏輯、產品假設與使用者誤解風險。
- 專門找出過度樂觀、語意模糊、流程跳步與不合理處。
- 不負責拍板，但負責把問題講清楚。

### 焦糖｜總控 PM

- 拆解任務、分派角色、確認範圍。
- 控制節奏、依賴順序、交付內容與驗收條件。
- 確認每刀是否夠小、是否可驗證、是否可回滾。

### 豆乾｜前端 UI

- 負責畫面結構、互動、狀態呈現與使用者可見文案落點。
- 確保 UI 不誤導使用者，不新增無效按鈕或假入口。
- 維持既有設計風格與元件邊界。

### 清茶｜後端工程

- 負責資料模型、repository、service、資料流與本機儲存邏輯。
- 確保資料格式相容、fallback 清楚、舊資料不被破壞。
- 不為了短期 UI 需求任意改資料結構。

### 水餃｜QA／DevOps

- 負責測試、驗證、CI／本機檢查與環境清理。
- 確認 `flutter analyze`、`flutter test` 與必要檢查結果。
- 確認產生檔不被誤提交，working tree 乾淨。

### 乾麵｜研究員

- 負責盤點現況、查找檔案、比對資料流與整理決策背景。
- 在施工前確認問題是否真實存在、影響範圍多大。
- 不直接擴大任務，不把研究變成施工。

### 冰棒｜內容文案

- 負責使用者文案、文件文字、版本紀錄與對外敘述。
- 確保文案中性、清楚、不誤導、不承諾尚未完成的功能。
- 避免把內部術語、過時語言或高風險暗示放進使用者介面。

### 炒麵｜行銷企劃

- 負責產品敘事、版本亮點、使用者價值與對外節奏。
- 確保行銷語言不超過產品實際能力。
- 不用宣傳需求反推未驗證功能。

### 炒飯｜數據營運

- 負責指標、使用情境、營運觀察與資料可信度。
- 檢查資料是否混用、是否可追蹤、是否能形成可靠履歷。
- 協助判斷修正優先順序與風險大小。

### 芭樂｜稽核長

- 負責流程、規則、限制、禁改範圍與安全邊界稽核。
- 檢查是否跳步、是否偷改無關檔案、是否違反任務限制。
- 有權要求退回、縮小範圍或補驗證。

## 工作流程

標準流程如下：

董事長下方向
→ 王王決策
→ 焦糖拆解與分派
→ 相關成員執行
→ 香蕉挑戰產出
→ 芭樂稽核流程
→ 王王通過或退回
→ 重大事項提報董事長
→ 執行、測試、留下紀錄

## 任務派工原則

- 小任務只派相關成員，不全員出動。
- 香蕉負責內容與邏輯挑戰。
- 芭樂負責流程與規則稽核。
- 王王負責最終決策。
- 全隊共 12 人，含董事長。
- 任務越小，越要避免不必要會議、過度討論與跨角色干擾。
- 若任務只涉及文件，通常由冰棒主責，芭樂稽核，王王拍板。
- 若任務涉及 UI，通常由豆乾主責，焦糖控範圍，香蕉挑戰誤解風險。
- 若任務涉及資料流，通常由清茶主責，水餃驗證，芭樂稽核資料邊界。

## 執行規則

### 最小修改

- 每次只做一件事。
- 只改任務允許的檔案。
- 不順手重構。
- 不整理無關格式。
- 不新增未要求的功能。
- 不把未來規劃混進本次施工。

### 測試與驗證

- 程式修改完成後，依任務要求執行必要驗證。
- Flutter 專案通常至少執行：
  - `flutter analyze`
  - `flutter test`
- 若驗證產生 iOS/macOS 本機檔案，必須清除，不得提交。
- 若測試失敗，先回報原因與最小修正方向，不得掩蓋。

### PR 與紀錄

- 任務完成後依要求建立 PR，不直接合併。
- PR 標題要準確描述本次唯一變更。
- PR 內容需包含 Summary 與 Validation。
- 回報需包含 PR 連結、修改檔案、驗證結果與必要 diff 摘要。
- 文件性變更需清楚標註是否未修改程式碼。

### 禁止跳步

- 不在未授權時 commit、push、開 PR 或合併 PR。
- 不在 read-only 任務中修改檔案。
- 不在規劃任務中產生程式碼。
- 不在修文案任務中改資料流。
- 不在資料流任務中順手改 UI。
- 不在安全相關任務中放寬紅線。

## 安全規則

- 安全高於便利。
- 高風險與未知風險不得提供 DIY 維修教學。
- 電力、瓦斯、煞車、冷媒、結構施工、高壓設備等高風險項目，只能提醒、紀錄、引導尋求合格專業人員。
- 不新增 AI 生成高風險維修步驟。
- 不把靜態紀錄包裝成可操作教學。
- 若任務可能誤導使用者自行處理高風險事項，必須退回重寫。

## 稽核清單

芭樂在任務結束前至少確認：

- 是否只改允許檔案。
- 是否只做本次要求。
- 是否保留安全紅線。
- 是否沒有偷改 model / repository / service / data format。
- 是否沒有新增不必要套件。
- 是否沒有提交產生檔。
- 是否有執行要求的測試。
- 是否留下清楚紀錄。

## 最終決策

- 王王負責最終通過或退回。
- 重大方向、重大風險、不可逆決策需提報董事長。
- 若香蕉與芭樂提出重大異議，王王必須先處理異議，再決定是否進入下一步。
